# Сітка і координати

## Координатна система

- Референс: **1920×1080 Figma px**
- Google Slides = 9 144 000 × 5 143 500 EMU
- `FPX = 9144000 / 1920 = 4762.5` EMU на Figma-піксель
- Конвертація шрифту: `pt × 2.667 = Figma px`
- **НІКОЛИ не використовувати PX=9525** (це 96dpi/960×540 — неправильна шкала)

## Сітка слайду (всі значення в Figma px)

```
W      = 1920   — ширина слайду
H      = 1080   — висота слайду
PAD    = 100    — мертва зона з усіх боків
UW     = 1720   — W - 2×PAD, робоча ширина
TH     = 100    — висота рядка заголовку
TG     = 100    — відступ між заголовком і бенто-картками
GAP    = 30     — відступ між бенто-картками
INN    = 30     — внутрішній відступ усередині картки з усіх 4 сторін
CY     = 300    — PAD + TH + TG, верхній край карток
CH     = 680    — H - PAD - CY, висота карток (для статичного bento_right_*)
TITLE_W= 1610   — LOGO_X - LOGO_GAP - PAD; ширина заголовкових боксів
                   (правий край = 1710, зазор до логотипа = 20px)
TITLE_GAP = 60  — фіксований відступ між низом ЗАГОЛОВОК і верхом ПІДЗАГОЛОВОК/ТЕКСТ
                   на ВСІХ композиціях
```

## INSET компенсація

`_INSET = 19px` — фіксований внутрішній відступ Google Slides (~0.25cm). REST API v1 не може його прибрати.

Усі TEXT_BOX отримують компенсацію через `makeElemTransform`:
```
box_x = text_x − 19,  box_y = text_y − 19
box_w = text_w + 38,  box_h = text_h + 38
```

Результат: `inner_width = box_w − 38 = text_w`, `inner_height = box_h − 38 = text_h`.

Word-fit і height-check рахуються за `inner_width`/`inner_height` (без INSET).

## Фіксовані позиції

```
_H1_FIXED_44 = 260px   — 44pt заголовки (cover, bento_right_*, section, badges)
_H1_FIXED_36 = 220px   — 36pt заголовки (title_body)
TITLE_GAP    = 60px    — відступ від заголовка до контенту нижче
_TITLE_W     = 1610px  — ширина заголовкових боксів
```

| Композиція | titleH | textY |
|---|---|---|
| `cover` | 260 | 420 |
| `section` / `section_red` | 260 | 420 |
| `title_body` | 220 | 380 |
| `bento_right_*` | 260 | 420 |
| `badges` | 220 | 380 |

## Інваріанти геометрії

1. **Елемент у межах**: кожен елемент повністю в `0–1920 × 0–1080` Figma px.
2. **Нема накладань**: body-текст не перекриває ряд карток; між ними відступ ≥ `GAP = 30px`.
3. **Зарезервована зона логотипа**: x∈[1730,1820], y∈[100,190] — жоден TEXT_BOX не перетинає. Заголовкові бокси використовують `TITLE_W=1610` (не `UW=1720`).

## Геометрія карток right bento (статична)

Висоти і Y-позиції `bento_right_*` визначаються сіткою (не адаптивно):

```
masterH = floor((_RBH - (n-1) × _GAP) / n)
```

| Композиція | Висота картки | Y старт |
|---|---|---|
| `bento_right_2x2` | `floor((_RBH - _GAP) / 2)` | `_PAD = 100` |
| `bento_right_2` | `masterH`; остання: `_RBH - (n-1)×(masterH+_GAP)` | `_PAD = 100` |
| `bento_right_3` | `masterH`; остання: `_RBH - (n-1)×(masterH+_GAP)` | `_PAD = 100` |

**Інваріант**: нижній край останньої картки завжди = `H - PAD = 980`. Остання картка поглинає залишок від `floor()`.
