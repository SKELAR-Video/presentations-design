# Бенто-картки

## Константи Right Bento

```
RBW = 860   — ширина bento-колонки праворуч
RBH = 880   — висота (H - 2×PAD)
RBX = 960   — PAD + LTW + GAP
LTW = 830   — ширина лівої текстової зони (UW - RBW - GAP)
```

Ліва зона: ЗАГОЛОВОК (44pt, 830×dynamic) + ТЕКСТ (22pt, 830×dynamic) — ТЕКСТ плаває на TITLE_GAP=60px нижче фактичного низу ЗАГОЛОВОК.

**Порожній ТЕКСТ-бокс**: навіть якщо слот відсутній — бокс переміщується нижче ЗАГОЛОВОК (h=1, невидимий). Без цього майстер-бокс залишається на y=390 і перетинає великий заголовок → `bento_left_overlap` FAIL.

## Скруглення кутів

Google Slides REST API **не дозволяє** задати фіксований радіус для `ROUND_RECTANGLE`.

Симуляція: `RECTANGLE` + 4 кути × (квадрат кольору фону + еліпс кольору картки).  
**Радіус: R=30px**, фіксований незалежно від розміру картки.

## Ширина карток

```
cw = floor((UW - (n-1) × GAP) / n)
```

| n карток | cw (px) |
|---|---|
| 1 | 1720 |
| 2 | 845 |
| 3 | 553 |
| 4 | 407 |

Ряд завжди від лівого поля (x=PAD) до правого (x=PAD+UW).

## Шрифт — єдиний pt для групи

Стратегія: **старт від role max**, зменшення якщо текст не влазить.

| Композиція | Role max |
|---|---|
| `two_columns` | 48pt |
| `three_columns` | 28pt |
| `bento_right_2` | 36pt |
| `bento_right_3` | 22pt |
| `bento_right_2x2` | 22pt |
| `kpi_cards` ЗНАЧЕННЯ | 48pt |

По кожній картці знаходимо максимальний pt, при якому текст вміщується. Фінальний груповий pt = **мінімум із них** (найтісніший текст диктує). Застосовується однаковий pt до всіх карток групи.

`updateTextStyle` виконується **після** `replaceAllText`, не до.

Перевірка: `textFitsParagraphs(text, dims.w, dims.h, pt)` — враховує і word-fit, і висоту.  
Word-fit guard і коефіцієнти → `docs/rules/typography.md`.

## Вертикальне позиціонування (нижній бенто)

Застосовується до `kpi_cards`, `two_columns`, `three_columns`.  
**НЕ застосовується** до `bento_right_*` — в них інша логіка (static grid, `_RBH=880`).

**Правила:**
- Нижній край карток = **980** (H − PAD) — завжди, картки «прилипають» донизу.
- Верх за замовчуванням = **540** (H/2) → мінімальна висота групи = **440px**.
- Якщо контент не вміщується у 440px → верх піднімається вище центру (більша cardH).
- Жорстка підлога: верх ніколи не вище `CY=300`.
- Вільний простір — **тільки зверху карток**, не всередині.

**Формула** (`kCY` = top of cards):
```
contentCardH   = estimated text height + 2×INN + 2×VERT_PAD
desiredTopY    = 980 − max(contentCardH, 440)
kCY            = max(desiredTopY, 300)   // hard floor
cardH          = 980 − kCY
```

Реалізація: `computeKpiAdaptive()` для `kpi_cards`, `buildBentoRowLayoutRequests()` для `two_columns`/`three_columns`.

## Висота картки — за вмістом

`cardH = contentH + 2 × VERT_PAD`

- `VERT_PAD = 40px` для бенто-рядів (`two_columns`, `three_columns`)
- `KPI_VERT_PAD = 30px` для `kpi_cards`
- Усі картки ряду — **однакова висота** (= найвища з них)
- **НЕ розтягувати** картку на весь слайд
- **НЕ розганяти** вміст штучно (`spaceAbove/Below`) — залишки простору знизу коротшої картки

## kpi_cards — tight group

ЗНАЧЕННЯ безпосередньо над ПІДПИСОМ, без пропорційного розподілу:
```
ЗНАЧЕННЯ box:  y = kCY + INN + KPI_VERT_PAD,           h = valH
ПІДПИС box:    y = kCY + INN + KPI_VERT_PAD + valH,    h = lblH
cardH          = valH + lblH + 2×INN + 2×KPI_VERT_PAD
```

kpi_cards геометрія майстра (Figma виміряно):
```
kCY0 = 356  = PAD + TH + subH + TG
kCH0 = 624  = H - PAD - kCY0
kw   = 407  = (UW - 3×GAP) / 4
```

Шрифт ЗНАЧЕННЯ: scale `[48, 36, 28, 22, 18, 14]` pt — найбільший, при якому текст ≤ 3 рядки.

## Буліти

Якщо вміст картки — список пунктів → кожен пункт з нового рядка з маркером `•`.

Автоматична конвертація (`preprocessBentoText`):
- `«пункт1 · пункт2»` → `«• пункт1\n• пункт2»`
- `«рядок1\nрядок2»` (не value+label) → `«• рядок1\n• рядок2»`

**Виняток**: цифра+підпис (`$5M\nнові клієнти`) — не конвертується.

## two_columns_labeled — підпис колонки (ПІДПИС_N)

`ПІДПИС_1`/`ПІДПИС_2` — мітка-категорія над колонкою, окремий слот (тільки в `two_columns_labeled`).

- Висота зони мітки = 89px (від y=451 до y=540, де починається `КОЛОНКА_N`) — не 50px з майстра (той розмір нецентральний, лишений для сумісності з `create-master`).
- Кегль підбирається автоматично через загальний auto-shrink цикл (`FONT_STEPS = [22, 18, 14, 10]`), обмежений floor'ом і цією реальною висотою 89px — не фіксований 22pt.
- `max_chars = 90` — відкалібровано під 2 рядки при 10pt у колонці ~835px.
- При переході в композицію без окремого слоту під підпис (варіанти `two_columns`/`two_columns_plain`/`two_columns_timeline`/`bento_right_2`) — текст підпису приклеюється до `КОЛОНКА_N`/`КАРТКА_N` як `"Підпис — тіло"`, не губиться. Деталі → `docs/rules/compositions.md`.

## Авто-нумерація (bento_right_* з числами)

Числові бенто (`bento_right_2_num`, `bento_right_3_num`) — додаткові константи для розміру числа та відступів:
```
_NUM_PAD      = 11    — відступ від краю картки до числового боксу
                        (INN - _INSET = 30 - 19 = 11, щоб visual top = visual left = 30px)
_NUM_H        = 100   — висота числового боксу (2 картки)
_NUM_GAP      = 30    — відступ між числом і текстом
_NUM_FONT_PT  = 37    — кегль числа (2 картки)
_NUM_H_3      = 70    — висота числового боксу (3 картки)
_NUM_GAP_3    = 20    — відступ (3 картки)
_NUM_FONT_PT_3= 26    — кегль числа (3 картки)
```

Для 3+ карток автоматично використовуються менші значення (`_NUM_FONT_PT_3`, `_NUM_H_3`).
