# Типографіка

## Шрифт

- **Inter, weight 500** — скрізь і завжди, без винятків.
- **`bold: false`** — абсолютне правило (навіть для заголовків).
- Ієрархія — тільки через розмір кегля, не через жирність.
- **`autofitType: 'NONE'`** — у всіх TEXT_BOX без винятків (Google Slides API v1 не підтримує shrink-on-overflow при записі через API).

## Кроки розміру

Дозволені кроки баді-тексту: **14, 18, 22pt** — інші не вводити без узгодження.

Шкали для автоматичного підбору кегля:
| Елемент | Шкала |
|---|---|
| Заголовок (`ЗАГОЛОВОК` slot) | `[44, 40, 36, 32, 28]` pt |
| Section/closing title (no subtitle) | `[66, 54, 44, 36, 28, 22]` pt |
| Cover title only | `[66, 54, 44, 36, 28, 22]` pt |
| Бенто-картки | `[48, 36, 28, 22, 18, 14, 10]` pt (з обмеженням role max) |
| KPI ЗНАЧЕННЯ | `[48, 36, 28, 22, 18, 14]` pt |

## Word-fit guard

Жодне ціле слово не має виходити за межі ширини боксу.

```
longestWordPx(text, pt) = max(word.length) × pt × 2.667 × CHAR_W
```

- **`CHAR_W = 0.65`** — коефіцієнт ширини символу (Inter Medium, Cyrillic-safe)
- **`SAFETY = 1.1`** — запас безпеки *(було 1.2; зменшено для кращого використання ширини)*
- Умова проходження: `longestWordPx × 1.1 ≤ inner_width`
  де `inner_width = element_width − 2 × _INSET` (_INSET = 19px)
- Пунктуація стрипується з країв слів перед вимірюванням: "активність," → 10 символів, не 11

Логування:
```
[word-fit] <label>: longest_word_len=N | est=NNN | est×1.1=NNN | inner_width=NNN | font=NN → PASS/FAIL
```

## Height check

```
cpl = floor(inner_width / (pt × 2.667 × CHAR_W))   — chars per line
lines = word-wrap simulation over paragraphs
text_height = lines × LINE_H(pt)
```

- **`LINE_H(pt) = pt × 2.667 × 1.4`** — висота рядка в Figma px
- Умова: `text_height ≤ inner_height` де `inner_height = element_height − 2 × _INSET`

Логування:
```
[bento-height] <label>: lines=N | text_height=NNN | inner_height=NNN | font=NN → PASS/FAIL
```

## Заголовки — кроки підбору

Кроки `[44, 40, 36, 32, 28]` pt. Перший, при якому `longestWordPx × 1.1 ≤ _LTW` (де `_LTW = 830px`) — обраний кегль.

`estimateLineCount` з фактором 0.48 використовується **лише** для розрахунку позицій (layout positioning), не для overflow detection.

Фіксовані висоти заголовкових боксів:
| Кегль / Слайд | titleH | textY |
|---|---|---|
| 44pt (cover, bento_right_*, section, badges) | 260px | 420px |
| 36pt (title_body, badges adaptive) | 220px | 380px |

`TITLE_GAP = 60px` — фіксований відступ від фактичного низу ЗАГОЛОВОК до верху ПІДЗАГОЛОВОК/ТЕКСТ на всіх композиціях.

## NBSP — нерозривний пробіл

Короткі слова (1–4 символи: `у`, `і`, `та`, `на`, `до`, `від`, `як`, `це` тощо) не висять у кінці рядка: `addNbsp()` замінює пробіл після них на U+00A0.

- Числові токени — не чіпати.
- `content_integrity` вважає U+00A0 ≡ звичайному пробілу.

## Крапка наприкінці — стрипування

`stripTrailingPeriod()` при рендері прибирає фінальну `.` у:
- `ЗАГОЛОВОК` — всі композиції
- Всі `КАРТКА_N` / `КОЛОНКА_N` — бенто-картки

`?`, `!`, `…`, `...` — **залишати**.

`content_integrity` проходить автоматично (stripped ⊂ original).

## Кольорування тексту з «:»

Правило діє на **всі не-заголовкові текстові слоти** (body, caption, підзаголовки) у всіх композиціях:
- Частина до `:` включно → колір **WHITE** (`#FFFFFF`)
- Частина після `:` → MUTED (`#A2A6B1`) на темному тлі; PINK (`#FCCACA`) на червоному тлі
- `ЗАГОЛОВОК` — завжди виключений

Кольорові токени → `docs/rules/theme-brand.md`.

## Текст не обрізати

**Ніколи** не додавати «…» і не вкорочувати контент мовчки. Якщо текст не влазить — це FAIL валідатора; виправляти треба в промпті LLM або в самому ТЗ, а не в коді обробки.
