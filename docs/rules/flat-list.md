# Правило «Плоский список = 1 слайд»

## Жорстке правило

**Плоский список завжди рендериться на ОДНОМУ слайді.** Ніколи не розбивати на кілька слайдів, не дублювати заголовок, не запихати в бенто.

## Логіка вибору композиції

| Тип контенту | Композиція | Слот |
|---|---|---|
| Плоский список **коротких** міток (1–3 слова, ≤20 символів) | `badges` | `ПУНКТИ` = мітки через `\n` |
| Плоский список **довгих** пунктів (фрази/речення) | `title_body` | `ТЕКСТ` = пункти через `\n` |
| 2+ іменовані групи з підпунктами | `bento_right_*` | картки |
| Числові метрики | `kpi_cards` | картки |

## Композиція `badges`

**Де:** `lib/compositions.ts` → `id: 'badges'`

**Слоти:**
- `ЗАГОЛОВОК` — заголовок (стандарт, угорі-зліва)
- `ПУНКТИ` — рядки через `\n`, кожна мітка ≤20 символів

**Рендер:** `buildBadgesRequests` у `lib/google.ts`
- ПУНКТИ placeholder видаляється, замість нього — pill-бейджі (ROUND_RECTANGLE + TEXT_BOX)
- Бейджі течуть рядками зліва-направо, переносяться на новий рядок автоматично
- Висота бейджа: ~103px (V_PAD=30 + line=43 + V_PAD=30)
- Горизонтальний паддинг: 30px; вертикальний паддинг: 30px
- Гep між бейджами: 16px по горизонталі, 16px між рядками
- Колір фону: `#1A1F2E` (CARD); текст: білий; шрифт: Inter 500, 18pt

**Майстер-слайд:** для роботи треба регенерувати майстер-дек через `/api/create-master`.

## Валідаторні перевірки (додані в `lib/validator.ts`)

| Перевірка | Коли FAIL |
|---|---|
| `no_literal_asterisk` | Будь-який слот містить `*` |
| `no_duplicate_title` | ЗАГОЛОВОК поточного слайду = ЗАГОЛОВОК попереднього |
| `badge_item_max_chars` | Будь-яка мітка в ПУНКТИ > 20 символів |

## Заборони для LLM

- НЕ розбивати плоский список на кілька слайдів
- Не дублювати ЗАГОЛОВОК між сусідніми слайдами
- Ніколи не вставляти `*` у жоден слот
- `badges` → лише для коротких міток; довгі пункти → `title_body`
