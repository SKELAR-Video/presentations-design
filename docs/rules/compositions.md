# Каталог композицій

## Вибір композиції — таблиця

| Тип контенту | Композиція | Ключовий слот |
|---|---|---|
| Обкладинка з заголовком + підзаголовком | `cover` | `ЗАГОЛОВОК`, `ПІДЗАГОЛОВОК` |
| Обкладинка тільки з заголовком | `cover_title_only` | `ЗАГОЛОВОК` |
| Розділювач / анонс теми | `section` | `ЗАГОЛОВОК` (+ opt. `ПІДЗАГОЛОВОК`) |
| Розділювач на червоному | `section_red` | `ЗАГОЛОВОК` (+ opt. `ПІДЗАГОЛОВОК`) |
| Завершальний слайд | `closing` | `ЗАГОЛОВОК` (+ opt. `ПІДЗАГОЛОВОК`, `ЗОБРАЖЕННЯ_1`) |
| Текстовий слайд | `title_body` | `ЗАГОЛОВОК`, `ТЕКСТ` |
| Плоский список **коротких** міток (≤20 символів) | `badges` | `ПУНКТИ` |
| Числові метрики | `kpi_cards` | `КАРТКА_N_ЗНАЧЕННЯ`, `КАРТКА_N_ПІДПИС` |

| Текст ліворуч + 2 бенто-картки праворуч | `bento_right_2` | `ЗАГОЛОВОК`, `ТЕКСТ`, `КАРТКА_1`, `КАРТКА_2` |
| Текст ліворуч + 3 бенто-картки праворуч | `bento_right_3` | `ЗАГОЛОВОК`, `ТЕКСТ`, `КАРТКА_1..3` |
| Текст ліворуч + 2×2 бенто-картки праворуч | `bento_right_2x2` | `ЗАГОЛОВОК`, `ТЕКСТ`, `КАРТКА_1..4` |

## Правило: плоский список = 1 слайд

**Плоский список завжди рендериться на ОДНОМУ слайді.** Ніколи не розбивати на кілька слайдів, не дублювати заголовок, не запихати в бенто.

- Плоский список **коротких** міток (1–3 слова, ≤20 символів) → `badges` (слот `ПУНКТИ`)
- Плоский список **довгих** пунктів (фрази/речення) → `title_body` (слот `ТЕКСТ`)
- 2+ іменовані групи з підпунктами → `bento_right_*`
- Числові метрики → `kpi_cards`

## Специфіка кожної композиції

### cover / cover_title_only
- Отримують випадковий фотофон (Mountain 0–5 з `public/assets/backgrounds/`).
- `cover_title_only` — тільки `ЗАГОЛОВОК`, без підзаголовка і дати з документа. Макет: SKELAR Logo.png (wordmark) праворуч вгорі; дата-пілюля ліворуч вгорі; ЗАГОЛОВОК по центру кадру. Детально → `docs/rules/images.md`.

### section / section_red / closing
- `section` автоматично отримує червоний фон `#FD3433` та білий логотип.
- `closing` без `ПІДЗАГОЛОВОК` і `ЗОБРАЖЕННЯ_1` рендериться ідентично `cover_title_only`.
- Динамічний кегль заголовка при відсутньому `ПІДЗАГОЛОВОК`: кроки `[66, 54, 44, 36, 28, 22]pt`.

### title_body
- `ЗАГОЛОВОК` 36pt, `ТЕКСТ` 18–22pt (word-fit визначає кегль).
- Заголовок: 220px висота. Текст починається через `TITLE_GAP=60px` нижче фактичного низу заголовку.

### badges
- `ПУНКТИ` — рядки через `\n`, кожна мітка ≤ 20 символів.
- Рендер (`buildBadgesRequests`): ROUND_RECTANGLE pill + TEXT_BOX для кожного пункту.
- Бейджі течуть рядками зліва-направо, переносяться автоматично.
- Розміри: висота бейджа ~103px (V_PAD=30 + line=43 + V_PAD=30), горизонтальний паддинг 30px.
- Колір фону: `#1A1F2E` (CARD); текст білий; Inter 500 18pt.
- Перед роботою треба регенерувати майстер-дек через `/api/create-master`.

### kpi_cards
- `КАРТКА_N_ЗНАЧЕННЯ` — ВИКЛЮЧНО числові значення (`+42%`, `$5M`, `×2`, `< 0.5%`).
- Речення або список у `КАРТКА_N_ЗНАЧЕННЯ` = FAIL.
- `ТЕКСТ` ≤ 70 символів (короткий субтитул, НЕ абзац).
- Кількість карток = кількість реальних метрик; порожні картки = FAIL.

### two_columns / three_columns
- Кожна `КОЛОНКА_N` — окрема бенто-картка в ряду.
- Шрифт і геометрія → `docs/rules/bento.md`.

### bento_right_2 / bento_right_3 / bento_right_2x2
- `ТЕКСТ` — лівий баді-текст поруч із картками; може бути відсутнім.
- Порожній `ТЕКСТ`-бокс все одно переміщується нижче `ЗАГОЛОВОК` (h=1, невидимий) — без цього майстер-бокс залишається на y=390 і перетинає заголовок → `bento_left_overlap` FAIL.

## Варіанти дизайну (expandPlanWithVariants)

`VARIANT_GROUPS` визначає, які композиції вважаються варіантами одного контенту:
```ts
[['two_columns', 'bento_right_2'], ['three_columns', 'bento_right_3']]
```

При генерації кожна відповідна композиція автоматично розширюється у N послідовних слайдів-варіантів (суфікси `_v1`, `_v2` тощо).

**Захист від втрати контенту**: варіант генерується лише якщо `remapSlotsForVariant` успішно переносить **усі** ненульові слоти. Якщо хоч один слот не може перенестись (напр. `ТЕКСТ` є у `bento_right_2`, але відсутній у `two_columns`) — варіант пропускається.

## Заборони для LLM

- НЕ розбивати плоский список на кілька слайдів.
- Не дублювати `ЗАГОЛОВОК` між сусідніми слайдами.
- Ніколи не вставляти `*` у жоден слот (маркдаун-залишки).
- `badges` → лише для коротких міток ≤20 символів; довгі пункти → `title_body`.
- `kpi_cards` → лише числові метрики; text, список, речення → `title_body` або `bento_right_*`.
