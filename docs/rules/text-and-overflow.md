# Text Layout — Anchored Grow-to-Fit Model

## Принцип

Кожен текстовий бокс має **фіксований якір** (верхній-лівий кут) і **межі росту** (`max_w`, `max_h`).
Бокс **росте від якоря** вправо/вниз до вмісту, але не виходить за межі.
**Обрізати текст — лише в крайньому разі**, коли навіть `max_w × max_h` не вистачає.

```
anchor ┐
       ├─────────── max_w ──────────────┐
       │  text flows here               │
       │                                │
       └────────────────────────────────┘ ← max_h (grows down from anchor)
```

## Поля CompositionSlot

| Поле | Тип | Опис |
|---|---|---|
| `anchor` | `{ x, y }` | Фіксована верхньо-ліва точка в Figma px. Не рухається. |
| `max_w` | number | Максимальна ширина від якоря (px). Не заходить у зарезервовані зони. |
| `max_h` | number | Максимальна висота від якоря (px). Після цього — обрізка (max_chars). |
| `float_after` | string | Назва слоту, нижній край якого є динамічним якорем для Y цього боксу. |
| `float_gap` | number | Відступ між нижнім краєм `float_after` і верхнім краєм цього боксу. |
| `max_chars` | number | Жорсткий ліміт символів — **останній рубіж**, не перший. |

## Пріоритет при переповненні

```
1. Розрости бокс по ширині (до max_w)
2. Розрости бокс по висоті (до max_h від якоря)
3. Зменшити шрифт (font-size steps: 22→18→14→10pt) — якщо ще не вміщається
4. max_chars — тільки якщо навіть на min-font у max_w×max_h не влазить
   НІКОЛИ не обрізати мовчки («…»)
```

## Зарезервовані зони

| Зона | Координати (Figma px) | Правило |
|---|---|---|
| Поля слайда | x < 100 або x+w > 1820; y < 100 або y+h > 980 | Жоден бокс не виходить за PAD=100 |
| Логотип SKELAR (top-right) | x ∈ [1730,1820], y ∈ [100,190] | Текстові бокси не перекривають логотип |
| Логотип SKELAR (bottom-left, bento_right_*) | x ∈ [100,190], y ∈ [890,980] | Аналогічно |

### TITLE_W — ширина заголовкових боксів у верхній смузі

```
LOGO_X   = W − PAD − LOGO_W = 1920 − 100 − 90 = 1730
LOGO_GAP = 20   (мінімальний відступ між правим краєм тексту і лівим краєм логотипа)
TITLE_W  = LOGO_X − LOGO_GAP − PAD = 1730 − 20 − 100 = 1610
```

**Правило**: будь-який TEXT_BOX у верхній смузі (y ≈ PAD=100) — ЗАГОЛОВОК, ПІДЗАГОЛОВОК, ДАТА cover — має `w = TITLE_W = 1610`, а не `UW = 1720`.

- Правий край тексту: `PAD + TITLE_W = 100 + 1610 = 1710`
- Зазор до логотипа: `1730 − 1710 = 20px` ✓
- Виняток: `bento_right_*` — тут логотип у нижньому-лівому куті, ЗАГОЛОВОК вже має `w=LTW=830` → без змін.

**Реалізовано у**:

| Файл | Що зроблено |
|---|---|
| `app/api/create-master/route.ts` | Константа `TITLE_W=1610`; всі ЗАГОЛОВОК-бокси у верхній смузі → `w=TITLE_W` |
| `lib/google.ts` | `_TITLE_W=1610`; `buildCoverFloatRequests` використовує `_TITLE_W` замість `_UW` |
| `lib/validator.ts` | Перевірка `logo_overlap`: TEXT_BOX не перетинається із зоною логотипа (FAIL = перекриття bounding box) |

## Приклад: cover

### До (старий майстер)
```
ЗАГОЛОВОК  x=100  y=100  w=1720  h=320  (фіксована висота)
ДАТА       x=100  y=928  w=500   h=52   ← ❌ нижній-лівий кут, вузький
```

### Після (нова модель)
```
# compositions.ts
{ name: 'ЗАГОЛОВОК', anchor:{x:100,y:100}, max_w:1720, max_h:400 }
{ name: 'ДАТА',      anchor:{x:100,y:100}, max_w:1720, max_h:80,
  float_after:'ЗАГОЛОВОК', float_gap:30 }

# Приклад рендеру (заголовок 1 рядок = 164px)
ЗАГОЛОВОК  x=100  y=100  w=1720  h=164  (обрізано до реального вмісту)
ДАТА       x=100  y=294  w=1720  h=52   ← ✅ (100 + 164 + 30)

# Приклад рендеру (заголовок 2 рядки = 328px)
ЗАГОЛОВОК  x=100  y=100  w=1720  h=328
ДАТА       x=100  y=458  w=1720  h=52   ← ✅ (100 + 328 + 30)
```

## Типографіка — нерозривні пробіли (висячі короткі слова)

### Правило

Короткі літерні слова (1–4 символи) — прийменники, сполучники, частки типу `у`, `і`, `з`, `та`, `на`, `до`, `для`, `від`, `як`, `це` тощо — **не повинні залишатися в кінці рядка**. Пробіл ПІСЛЯ такого слова замінюється на нерозривний (U+00A0), і слово переноситься на новий рядок разом із наступним.

### Алгоритм (`addNbsp` у `lib/google.ts`)

```
Для кожного слова W у тексті:
  якщо len(W) ∈ [1,4] AND W складається ТІЛЬКИ з літер (укр/лат) AND після W є пробіл AND після пробілу є наступне слово:
    замінити пробіл після W на U+00A0
```

**Винятки:**
- Слово містить цифру або символ → **не чіпати** (числові токени: `2M`, `10K`, `2024` тощо)
- Слово є останнім у тексті (немає пробілу після нього) → нічого не робити
- Слово є фрагментом довшого слова → негативний lookbehind `(?<![А-ЯЁІЇЄҐа-яёіїєґA-Za-z0-9])` запобігає хибним спрацюванням

**Застосовується до:** заголовків, підзаголовків, body, булітів, підписів — тобто до **всіх** текстових слотів.  
**Не застосовується до:** image-слотів, порожніх слотів.

### Узгодження з `content_integrity`

NBSP — дозволена відображувальна трансформація. Перед дослівною перевіркою рядок нормалізується:

```ts
const normalized = line.replace(/ /g, ' ')
const verbatimOk = sourceText.includes(normalized)
```

Заміна `пробіл → NBSP` = **PASS**, не «текст змінено».

### Де реалізовано

| Файл | Що робить |
|---|---|
| `lib/google.ts` | `addNbsp(text)` — замінює пробіл після коротких слів на U+00A0 |
| `lib/google.ts` | виклик `addNbsp(replaceText)` у циклі `replaceAllText` (Step 5) |
| `lib/validator.ts` | `checkContentIntegrity` — нормалізує U+00A0 → пробіл перед `sourceText.includes(...)` |

### Крапка наприкінці заголовка

**Правило**: якщо слот `ЗАГОЛОВОК` закінчується на `.` — крапку прибрати при рендері.

| Кінцевий символ | Дія |
|---|---|
| `.` (крапка) | прибрати |
| `?` або `!` | залишити — змістовна пунктуація |
| `…` (U+2026) або `...` | залишити — три крапки є частиною тексту |

Застосовується до слота `ЗАГОЛОВОК` та **всіх бенто-карток** (`КАРТКА_N`, `КОЛОНКА_N`). Body (`ТЕКСТ`), `ПІДЗАГОЛОВОК`, `ПІДПИС` — не чіпати.

Для бенто-карток із роздільником ` · ` крапка знімається з кожного item ще до конвертації в буліт-рядки (`preprocessBentoText`).

**Content-integrity**: рядок без кінцевої крапки завжди є підрядком оригіналу → `sourceText.includes(stripped)` = `true` автоматично. Жодних змін у валідаторі не потрібно.

**Де реалізовано**: `lib/google.ts` → `stripTrailingPeriod()`, виклик у `preprocessBentoText` (per-item) та у циклі `replaceAllText` (цілий слот).

### Фіксований відступ TITLE_GAP = 60px (всі композиції)

**Правило**: верх `ПІДЗАГОЛОВОК` (або `ТЕКСТ`, якщо він є "підзаголовком") завжди = низ `ЗАГОЛОВОК` + 60px, незалежно від кількості рядків заголовка.

```
titleH = estimateLineCount(title, w, pt) × h1LineH + 4  (max 300–400px)
subY   = PAD + titleH + 60
```

| Композиція | Slot ЗАГОЛОВОК (pt) | Slot нижче | Реалізовано в |
|---|---|---|---|
| `cover`          | 44pt, w=1610 | `ПІДЗАГОЛОВОК` | `buildCoverFloatRequests` |
| `section` / `section_red` | 44pt, w=1610 | `ПІДЗАГОЛОВОК` | `buildSectionFloatRequests` |
| `title_body`     | 36pt, w=1610 | `ТЕКСТ`         | `buildTitleBodyFloatRequests` |
| `bento_right_*`  | 44pt, w=LTW=830 | `ТЕКСТ`      | `buildBentoRightLeftColumnRequests` |

Коефіцієнт для висоти рядка: `h1LineH = round(pt × 2.667 × 1.08)` — точніший за глобальний `lineH` (1.4), враховує lineSpacing=90 майстра.

**compositions.ts**: кожен слот нижче ЗАГОЛОВОК матиме `float_after: 'ЗАГОЛОВОК', float_gap: 60` для документування.

#### Порожній secondary-слот — обов'язково переміщуємо, h=1

Якщо `ПІДЗАГОЛОВОК`/`ТЕКСТ` відсутній у плані — майстер-бокс все одно залишається у DOM (просто порожній після `replaceAllText`). Якщо не перемістити — він стоїть на фіксованій y=390 і перетинає ЗАГОЛОВОК, коли той займає більше 290px.

**Правило**: float-функції завжди емітують `makeElemTransform` для secondary-боксу — навіть якщо слот порожній. Висота: `bodyText ? textMaxH : 1` (h=1 = невидимий, але не перетинає заголовок).

```ts
// Завжди, незалежно від наявності тексту:
if (raw.includes('{{ТЕКСТ}}')) {
  reqs.push(makeElemTransform(el.objectId, x, textY, w, bodyText ? textMaxH : 1, sW, sH))
}
```

**Де реалізовано**: `buildBentoRightLeftColumnRequests`, `buildSectionFloatRequests`, `buildTitleBodyFloatRequests`.  
**Валідатор**: `bento_left_overlap` — перевіряє будь-які боксові перетини, навіть порожні.

### bento_right — ліва колонка (ЗАГОЛОВОК → ТЕКСТ float)

В майстер-шаблоні ліва колонка `bento_right_*` має `ЗАГОЛОВОК` (44pt, h=260) і `ТЕКСТ` (22pt, y=390). Коли заголовок переноситься на кілька рядків — він виходить за межі боксу і перекривається з `ТЕКСТ`.

**Правило**: `ТЕКСТ` завжди розміщується суворо нижче `ЗАГОЛОВОК` з відступом `TITLE_GAP=60px`.

```
ЗАГОЛОВОК: x=PAD(100), y=PAD(100), w=LTW(830), h=dynamic (max 400px)
ТЕКСТ:     x=PAD(100), y=PAD + titleH + 60,    w=LTW(830), h=до лого − 20px
```

Висота `ЗАГОЛОВОК` розраховується точніше ніж глобальний `lineH`: коефіцієнт `1.08 = 1.2 × 0.9` (1.2 = normal line, 0.9 = lineSpacing=90 із майстра). Максимум — 400px (для 3-рядкових заголовків).

**Де реалізовано**: `lib/google.ts` → `buildBentoRightLeftColumnRequests()`, виклик після Cover adaptive у pipeline.

**Валідатор**: `bento_left_overlap` — перевіряє, що жоден лівий текстовий бокс не перетинається з наступним.

### bento_right — шрифт карток, багаторядковий текст

`pickBentoPt` обирає єдиний кегль для всього ряду. Раніше функція `textFits` ігнорувала `\n` (символ нового рядка), що призводило до недооцінки кількості рядків у буліт-списках і надто великого шрифту.

**Правило**: для бенто-карток — `textFitsParagraphs`: кожен рядок `\n` починає новий рядок, рядки підсумовуються.

Висота в `bentoDims` = `maxCardH − 2 × BENTO_VERT_PAD` (не `maxCardH − 2×INN`), що враховує внутрішній відступ карток при оцінці bounds колонки.

Для `bento_right_2x2` (4 вузькі картки, innerW=355px) при довгих буліт-списках кегль може опуститись до 10–14pt — це коректна поведінка (багато тексту в малих картках).

**Де реалізовано**: `lib/google.ts` → `textFitsParagraphs()`, `bentoDims()` (оновлена висота), `pickBentoPt()`.  
**Валідатор**: `checkBentoLayout` оновлено на `_vTextFitsParagraphs`, `_vBentoDims` синхронізовано.
